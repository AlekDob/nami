- [09:30] (Alek) fix: session mixing between cron jobs and user chat — 3 root causes: (1) two SessionStore instances writing to same index.json (cron overwrites user sessions), (2) single Agent instance with no mutex allows concurrent agent.run() calls, (3) onToolUse callback corrupted by concurrent callers. Fix: shared SessionStore singleton, promise-based run queue on Agent, per-call onToolUse via RunOptions. Deployed to server.

No API key needed. Includes surrogate pair safe truncation. (2) Installed `@fabriqa.ai/youtube-transcript-mcp` as MCP server for Quack/Claude Code. Now Nami can "read" YouTube videos when given a link.
