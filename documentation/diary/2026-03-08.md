- [12:33] (Alek) fix: YouTube transcript — switched from npm library proxy to yt-dlp via Mac agent. Previous approach (`youtube-transcript-plus` with `macProxyFetch` curl proxy) had 3 issues: (1) Mac agent exec maxBuffer 1MB too small for YouTube pages (~1.4MB), (2) `/file` endpoint restricted to `/Users/alekdob` path, (3) POST body shell escaping hell for Innertube player API. New approach: reuses same Python script as Quack `youtube-transcript` skill — runs `yt-dlp` on Mac via `osascript` through Mac agent `/exec`. Benefits: chapters, multi-language, auto/manual subtitle detection, no API keys. Also bumped Mac agent maxBuffer 1MB→5MB. Tested TED talk: 324 segments, 24 chapters, 50 languages available.
