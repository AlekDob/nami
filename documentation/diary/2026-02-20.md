- [09:00] (Alek) fix: WebSocket shows "offline" on macOS first launch — server now sends `{"type":"connected"}` on WS open, client handles it to confirm connection and update notch status immediately. Root cause: `isConnected` was only set to `true` on first *received message*, but server never sent anything until user chatted. Added `connected` to `WsServerMessage` type union + `WSIncoming` Swift enum.
- [09:30] (Alek) feat: notch panel response animation + click-to-open. When Nami responds: collapsed notch shows a blue pulse ring on the status dot; expanded view animates the response bubble in with scale+opacity spring. Clicking the response bubble opens the main NamiOS window (`NSApp.activate` + bring main window to front). Small arrow icon hint added to response bubble.
- [10:00] (Alek) feat: ASCII wave animation in notch shoulders. New `NotchWaveText` view with `TimelineView` renders `~∿≈~∿` chars with sinusoidal vertical offset. Two styles: `.thinking` (slow 0.12s, orange) and `.done` (fast 0.08s, green). Waves appear in both left and right shoulders of collapsed notch.
- [10:15] (Alek) fix: notch shoulders too narrow (40px) for wave animation — waves clipped by hardware notch. Dynamic shoulder width: 40px normal → 80px when waves active. Synced `shoulderWidth` in `NotchContentView` + `visibleMaskRect()` in `NotchPanelManager` via `isShowingWaves` flag. Mask expands with spring animation when thinking/done.
