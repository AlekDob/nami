- [14:00] (Alek) Added `POST /api/knowledge` endpoint to create brain entries via REST API. The internal `saveKnowledge()` method already existed but wasn't exposed as a route. Accepts title, content, summary, sourceType, tags[], optional sourceUrl/ogImage. Returns 201 with generated UUID.
- [14:00] (Alek) Created `nami-remote` skill (`~/.claude/skills/nami-remote/SKILL.md`) — full API reference for all ~30 REST endpoints + WebSocket. Enables any external agent (Quack, etc.) to interact with Nami's brain, jobs, memory, chat, soul, and all features via Tailscale at `100.81.200.26:3000`.
- [14:30] (Alek) Added `PUT /api/knowledge/:id` endpoint for updating existing brain entries. Required new `updateKnowledge()` method in both `indexer.ts` (SQLite UPDATE + FTS re-index + tag replacement + vector re-index) and `store.ts` (wrapper). Partial updates supported — only send fields you want to change. Without this, agents had to DELETE+POST to update entries.
